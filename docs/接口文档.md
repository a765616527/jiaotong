# 接口文档（实现版）

本文档对应当前后端代码实现版本。  
统一约束：仅使用 `GET` / `POST`，统一响应结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

## 1. 认证模块

## 1.1 登录

1. 方法：`POST`
2. 路径：`/api/auth/login`
3. 请求体：

```json
{
  "username": "admin",
  "password": "Admin@123456"
}
```

4. 返回：`token + user`

## 1.2 当前用户信息

1. 方法：`GET`
2. 路径：`/api/auth/me`
3. Header：`Authorization: Bearer <token>`

## 2. 事故管理

## 2.1 分页查询事故

1. 方法：`GET`
2. 路径：`/api/accidents`
3. 参数：
4. `pageNum`、`pageSize`
5. `roadName`（可选）
6. `accidentType`（可选）
7. `startTime` / `endTime`（可选，ISO DateTime）

## 2.2 新增事故

1. 方法：`POST`
2. 路径：`/api/accidents`
3. 请求体字段：
4. `occurTime`、`roadName`、`areaCode`
5. `longitude`、`latitude`
6. `accidentType`、`vehicleCount`、`casualtyCount`
7. `description`

## 2.3 编辑事故

1. 方法：`POST`
2. 路径：`/api/accidents/{id}/update`

## 2.4 删除事故

1. 方法：`POST`
2. 路径：`/api/accidents/{id}/delete`

## 2.5 事故详情

1. 方法：`GET`
2. 路径：`/api/accidents/{id}`

## 2.6 CSV导入

1. 方法：`POST`
2. 路径：`/api/accidents/import-csv`
3. Content-Type：`multipart/form-data`
4. 字段：`file`
5. 返回：
6. `success`、`failed`
7. `errors[]`（行号、原始内容、失败原因）

## 3. 风险与预警

## 3.1 手动触发风险评估

1. 方法：`POST`
2. 路径：`/api/risk/evaluate`
3. 返回：路段风险列表，符合条件会自动生成 `DRAFT` 预警

## 3.2 风险热点

1. 方法：`GET`
2. 路径：`/api/risk/hotspots`

## 3.3 预警分页

1. 方法：`GET`
2. 路径：`/api/warnings`
3. 参数：`pageNum`、`pageSize`、`status`

## 3.4 预警状态流转

1. 确认：`POST /api/warnings/{id}/confirm`
2. 发布：`POST /api/warnings/{id}/publish`
3. 解除：`POST /api/warnings/{id}/resolve`

## 3.5 批量流转

1. 批量确认：`POST /api/warnings/batch/confirm`
2. 批量发布：`POST /api/warnings/batch/publish`
3. 批量解除：`POST /api/warnings/batch/resolve`
4. 请求体：

```json
{
  "ids": [1, 2, 3]
}
```

## 3.6 处置记录

1. 方法：`POST`
2. 路径：`/api/warnings/{id}/disposal-records`
3. 请求体：

```json
{
  "actionDesc": "已完成夜间增设巡逻"
}
```

## 3.7 AI预警解释（单条）

1. 方法：`GET`
2. 路径：`/api/warnings/{id}/ai-explanation`

## 4. 统计模块

## 4.1 综合统计

1. 方法：`GET`
2. 路径：`/api/stats/overview`
3. 参数：`days`（7~30，超界自动修正）
4. 返回：趋势、类型占比、高发路段、预警状态占比、总数

## 5. 管理员模块

## 5.1 用户管理

1. 列表：`GET /api/admin/users`
2. 新增：`POST /api/admin/users`

## 5.2 规则管理

1. 列表：`GET /api/admin/rules`
2. 更新：`POST /api/admin/rules/update`

## 5.3 审计日志

1. 操作日志：`GET /api/admin/logs`
2. AI调用日志：`GET /api/admin/ai-logs`
3. 规则修改日志：`GET /api/admin/rule-change-logs`
4. 规则日志支持参数：`ruleKey`、`startTime`、`endTime`

## 5.4 管理汇总

1. 总览卡片：`GET /api/admin/summary`
2. 趋势分析：`GET /api/admin/summary/trend?days=7|30`
3. 趋势返回包含：
4. `dates`
5. `accidentCounts`、`warningCreatedCounts`
6. `loginCounts`、`loginFailCounts`
7. `compare`（当前周期 vs 上一周期对比）

## 5.5 字典管理（管理员）

1. 分页：`GET /api/admin/dicts`
2. 新增：`POST /api/admin/dicts`
3. 更新：`POST /api/admin/dicts/{id}/update`
4. 删除：`POST /api/admin/dicts/{id}/delete`

## 5.6 字典查询（业务侧）

1. 方法：`GET`
2. 路径：`/api/dicts/items`
3. 参数：
4. `dictType`（必填）
5. `enabled`（默认 1）

## 6. 大模型模块

1. 事故文本抽取：`POST /api/ai/extract-accident-fields`
2. 预警解释生成：`POST /api/ai/generate-warning-explanation`
3. 处置建议生成：`POST /api/ai/generate-disposal-suggestions`
4. 管理员问答：`POST /api/ai/assistant-query`

说明：
1. 未配置模型密钥时，接口返回规则兜底内容，不阻塞演示。
2. 模型调用过程会写入 `ai_call_log`。

## 7. 健康检查

1. 方法：`GET`
2. 路径：`/api/health`
3. 用途：服务存活探测
