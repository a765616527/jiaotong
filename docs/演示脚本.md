# 答辩演示脚本（建议版）

## 1. 演示准备

1. 启动后端：`cd server && mvn spring-boot:run`
2. 启动前端：`cd web && npm run dev`
3. 准备账号：
4. 管理员：`admin / Admin@123456`
5. 交通管理人员：在管理员页现场新建
6. 若数据不足，先生成并导入模拟数据：
7. `cd sql && ./gen_mock_data.sh 500 > mock_accidents_500.sql`
8. `mysql -uroot -p traffic_warning < mock_accidents_500.sql`

## 2. 演示主线（约 8~12 分钟）

## 2.1 登录与权限

1. 管理员登录，展示管理员菜单（含字典管理）
2. 退出后用交管员登录，展示受限菜单

## 2.2 字典与规则配置（管理员）

1. 进入“字典管理”，演示事故类型增改删查
2. 进入“规则配置”，修改阈值并保存
3. 在“规则修改历史”展示变更记录与筛选

## 2.3 事故管理（交管员）

1. 新增一条事故
2. 编辑一条事故
3. CSV 导入（展示成功/失败明细）
4. 演示“事故类型字典联动校验”

## 2.4 风险评估与预警闭环

1. 在“预警中心”点击“触发风险评估”
2. 展示 `DRAFT -> CONFIRMED -> PUBLISHED -> PROCESSING -> RESOLVED`
3. 演示发布/解除二次确认弹窗
4. 演示批量发布/解除与失败明细

## 2.5 可视化与特色功能

1. 地图分析页（点位/热力模式切换）
2. 统计报表（趋势、类型占比、路段 TOP、预警状态）
3. Three.js 三维风险态势页
4. 大模型能力页：抽取、解释、处置建议、问答

## 2.6 管理分析与审计

1. 管理员汇总卡片（用户、事故、预警、AI调用、登录）
2. 管理员趋势图（7天/30天切换）
3. 操作日志与 AI 调用日志展示

## 3. 讲解关键句（可直接复用）

1. “系统采用规则驱动完成预警判定，保证可解释性。”
2. “大模型只做增强，不参与最终触发判定，所有关键动作由人工确认。”
3. “预警全流程可追溯，支持日志审计与规则变更留痕。”
